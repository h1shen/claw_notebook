# 计划模式深度指南

🟡 个人实践

## 什么时候用计划模式

计划模式适用于任何复杂度超过"单步操作"的任务：

- 新功能开发（涉及多个文件）
- 架构重构
- 调试复杂问题（根因不明确时）
- 任何你不确定"从哪里开始"的任务

**核心原则：** 真正至关重要的第一步是思考和计划，而非直接开始编码。

## 如何触发

在 Claude Code 交互界面中，按 **Shift+Tab 连按两次** 进入计划模式。

计划模式的输出远胜于直接输入提示词。

## 如何引导出高质量计划

进入计划模式后，提供足够的背景信息：

**好的计划请求示例：**
```
我需要给现有的 Express 应用添加用户认证。
当前架构：Mongoose + Express，用户模型已存在。
要求：JWT token，7天过期，/api/auth 路由，保护 /api/protected/* 下所有路由。
请先给我一个实施计划，列出需要修改的文件和步骤。
```

**引导技巧：**
1. 说明现有架构和约束
2. 明确技术栈和不能改变的部分
3. 要求按文件列出修改步骤
4. 要求识别潜在风险点

## 计划执行技巧

拿到计划后：

1. **审查计划**：确认每个步骤合理，没有遗漏
2. **分批执行**：不要让 Claude 一口气执行所有步骤
3. **检查点验证**：每完成一个阶段，运行测试/检查输出
4. **保存计划**：让 Claude 把计划写入 SCRATCHPAD.md，方便跨会话继续

## 计划模式 vs 直接输入：对比案例

| | 直接输入 | 计划模式 |
|---|---|---|
| **请求** | "添加用户认证" | 先计划，再执行 |
| **Claude 行为** | 自由发挥，可能选择错误方案 | 先理解全局，再精准实施 |
| **结果质量** | 不稳定，可能需要大量返工 | 更可预测，更少返工 |
| **适用场景** | 简单单步任务 | 任何多步骤任务 |

## 两条建议

1. **开始学习**：即使每次只学一点，也必须开始积累规划能力
2. **与 AI 深度交流**：和 Claude 进行双向对话，而不是单向输入
