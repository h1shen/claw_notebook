# Claude Code 概览与核心理念

🔵 官方文档 | 🟡 个人实践

## 什么是 Claude Code

Claude Code 是 Anthropic 推出的 **命令行 AI 编程助手**，设计定位是 agentic coding assistant——能够主动探索代码库、执行命令、修改文件，而不只是被动回答问题。

### 与 Claude.ai 的核心区别

| | Claude.ai（网页版）| Claude Code（CLI）|
|---|---|---|
| **交互方式** | 对话框输入/输出 | 终端内，可直接操作文件 |
| **文件访问** | 需要手动粘贴代码 | 直接读写项目文件 |
| **命令执行** | 不能执行命令 | 可运行 shell 命令、测试、构建 |
| **上下文** | 手动管理 | 自动感知项目结构 |
| **自主程度** | 被动响应 | 主动探索、多步执行 |

### 设计哲学

Claude Code 的设计哲学是 **agentic**（主动代理）：

- 它不只是"帮你写代码"，而是作为你的编程伙伴参与整个开发流程
- 可以理解整个代码库的上下文，而不是只看一个文件片段
- 可以自主完成多步骤任务（读文件 → 理解 → 修改 → 运行测试 → 调整）

## 核心理念

### 1. 你在"编程"它，不是在聊天

> **你不是在和它聊天，你是在编程它。**

把 Claude Code 想象成一个非常聪明但需要明确指令的团队成员：

- 给它清晰的约束（技术栈、架构决策、不允许做的事）
- 提供背景信息（为什么这么做，不只是做什么）
- 用 CLAUDE.md 持久化这些约束

### 2. 计划先于执行

不要直接让它开始写代码。先计划：

- 用计划模式（Shift+Tab×2）生成实施方案
- 让它列出将要修改的文件和步骤
- 确认计划合理后再执行

### 3. 上下文是有限资源

即使有 200k token 的上下文窗口，性能在使用 20-40% 时就开始退化：

- 每个任务使用独立会话
- 及时清理无关上下文
- 用外部文件（SCRATCHPAD.md）跨会话保存进度

## 快速开始

### 安装

```bash
npm install -g @anthropic-ai/claude-code
```

### 启动

```bash
# 进入交互模式
claude

# 执行单个指令后退出
claude "解释这个函数的作用"

# 无头模式（适合脚本）
claude -p "审查这段代码" < code.py
```

### 首次配置建议

1. 创建项目级 `CLAUDE.md`，描述项目特有信息（见 [CLAUDE.md 编写法则](claude-md.md)）
2. 了解计划模式（见 [计划模式深度指南](plan-mode.md)）
3. 配置全局 `~/.claude/CLAUDE.md` 写入你的个人偏好

## 工作模式概览

| 模式 | 触发方式 | 适用场景 |
|------|---------|---------|
| 交互模式 | 直接 `claude` | 日常开发，对话式调试 |
| 计划模式 | Shift+Tab×2 | 复杂任务规划，多步骤任务 |
| 无头模式 | `claude -p "..."` | 自动化脚本，CI/CD 集成 |

## 当你被卡住时：改变方法，而非继续强推

识别死循环信号：Claude 尝试 → 失败 → 再尝试 → 再失败。

**四个改变方法的策略：**

1. **清空对话** - 累积的上下文可能正在迷惑它，全新开始可以解决问题
2. **简化任务** - 分解成更小的部分；复杂任务失败通常意味着初始计划不充分
3. **展示而非告知** - 亲手写一个最小化的正确示例，说"把这个模式应用到其他部分"
4. **重构问题** - 换一个角度描述；如果重复解释了三遍，是时候改变策略了
